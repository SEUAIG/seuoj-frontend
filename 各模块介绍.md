# React Router
1. 版本号^6.20.1
2. 作用是在单页应用实现页面导航与路由管理 无需每次请求新页面（局部刷新）
3. 具体如何使用可参考对应文档 这里只介绍功能与使用技巧
4. 注意点：
   - `BrowserRouter`作为路由上下文声明与使用在`main.tsx` ，这也是上下文相关声明的地方 后续应当遵守
   - 我们使用组件声明式的路由实现（`<Routes>`） 在最新版有配置对象式的实现`const routes = useRoutes（{}）` 但这种太复杂了 如果你有兴趣后续可改成这种 
   - 注意只有一个`Routes` 其余为`Route` 你可以使用自闭标签（`<Route/>`）与常规标签(`<Route><Route/>`)来声明 但嵌套路由的外层路由只能是常规标签 包裹子路由
   - 不要再使用`<a/>` 后续只使用`<Link/>` 注意这个与`<NavLink/>`区别 后者用于导航 会给当前使用的导航一个`active`的类名，但是`tailwindCss`不是面向类名的 所以我们的组件使用如下
     ```tsx 
     <NavLink to={to} end={end}>
       {({ isActive }) => (
         <div
           className={`
             flex items-center gap-1
             px-4 py-2 rounded-full
             transition-all duration-400 relative
             hover:bg-slate-300
             ${
               isActive
                 ? "bg-accent ring-1 ring-border text-foreground font-medium shadow"
                 : "text-muted-foreground hover:bg-muted/100 hover:text-foreground"
             }
           `}
         >
           <Icon
             strokeWidth={isActive ? 2.5 : 1.5}
             size={36}
             className="h-4 w-4"
           />
           <span
             className={`whitespace-nowrap ${
               isActive ? "font-bold" : "font-normal"
             }`}
           >
             {label}
           </span>
           {isActive && (
             <span
               className="
           absolute left-2 right-2 -bottom-1
           h-0.5
           rounded-full
           bg-primary
         "
             />
           )}
         </div>
       )}
     </NavLink>
     ```
    使用特殊技巧 当函数作为`<NavLink/>`的子元素时 会自动传入参数`isActive`来动态赋予CSS属性 `end`为`true`时 精确匹配才会赋予`active`属性
5. 实际作用 
   - 将特定的URL映射到对应的React组件
   - 管理浏览器历史记录 （nav（-1）是跳转上一个界面）
6. 使用的功能特性 
   - 嵌套路由
     ```tsx 
     <Route element={<AuthLayout />}>
       <Route path="/login" element={<LoginPage />}></Route>
       <Route path="/signup" element={<SignupPage />}></Route>
       <Route path="/forget" element={<ForgetPage />}></Route>
     </Route>
     ```
    当条件匹配时 子路由会渲染到父路由组件`<Outlet/>`对应的位置
    注意 这里外层`Route`没有给出路径 说明这是一个纯粹的父路由不会单独出现，只在url匹配其子路由时渲染
    若给出路径 则可能出现只渲染父路由的情况 此时`<Outlet/>`渲染为空
    `/`开头的`path`表示这是一个绝对路径，若没有则是相对路径（一般用在父路由给出路径的情况下）相对父路由
    - 路径导航
     ```tsx 
     <Route index element={<Navigate to="/home" />}></Route>
     const nav = useNavigate();
     nav(-1)
     ```
    以上分别是声明式导航 使用`<Navigate/>`组件 一旦渲染 立即跳转 与 函数式导航 使用`nav`函数导航
    - 动态路由
     ```tsx 
     <Route path=":id" element={<SubmissionPage/>}/>
     ```
    使用`：id`表示后续参数任意 但全部会作为`id`参数 可在组件内部使用`useParams`读取 用于传递信息

# Redux
1. 版本号^6.20.1
2. 目标是实现本地状态的管理，使得各组件可以直接读取本地状态，而不用逐个传递，避免prop drilling，我们实际使用的是`RTK`，一种对`redux`的简化，这也是官方推荐的。
3. 注意点：
   - 核心思想：**状态集中管理，单向数据流**。
   - 必须配置 Store、Slice 和 Provider 才能生效。
4. 使用流程（以我们代码为例）
   1. 创建store，在`src\app\store.ts`，所有状态都存在这里
      ```tsx
      // 创建 store
      export const store = configureStore({
          reducer: {
              // 这里注册不同的业务模块（Slice）
              // 例如：auth 模块负责管理用户登录状态 不同的模块具有不同的模块名和对应的reducer
              auth: authReducer, 
          }
      })
      // 导出类型，方便在组件中使用时有 TS 提示
      export type RootState = ReturnType<typeof store.getState>
      // 接受一个函数类型 返回这个函数的返回值类型 内置 类型相关用type 与 尖括号
      export type AppDispatch = typeof store.dispatch;
      // 想dispatch thunk 必须要有这一步
      ```
   2. 声明对应的Slice（切片）用于管理对应状态`src\features\auth\authSlice.ts`
      包括三大部分，初始状态，修改方法，异步操作 具体你可参考代码，但要注意以下几点，可能会在你写代码中碰到：
      - **定义异步操作 (Thunk)**：
        ```tsx
        // login.pending (进行中), login.fulfilled (成功), login.rejected (失败)
        export const login = createAsyncThunk(
          "auth/login", // 动作名称
          async ({ username, password }: LoginPayload, thunkAPI) => {
            // 发送网络请求... 函数只有两个默认参数 一个是{}传入参数 一个是thunkApi 注意传参数要打包传入
            const res = await fetch(`${ENV.API_BASE_URL}/api/auth/login`, ...);
            const result = await res.json();
            
            // 如果失败，返回错误信息
            if (result.code !== 200) {
              return thunkAPI.rejectWithValue("登录失败");
            }
            // 异步函数的返回值fulfilled会作为action payload rejected的error。message 会是reject with value
            return result; 
          }
        );
        ```
      - **创建 Slice**：
        ```tsx
        const authSlice = createSlice({
          name: "auth",
          initialState,
          // 同步修改状态的方法
          reducers: {
            resetAuth(state) {
              // 用于注销用户
              state.user = null;
              state.isAuthenticated = false;
            },
          },
          // 监听异步操作的状态变化
          extraReducers: (builder) => {
            builder
              .addCase(login.fulfilled, (state, action) => {
                state.isAuthenticated = true;
                const { jwt, username } = action.payload.data;
                state.jwt = jwt;
              })
              .addCase(login.rejected, (state, action) => {
                state.isAuthenticated = false;
                state.error = action.payload as string;
              });
          },
        });
        // 导出同步 action
        export const { resetAuth } = authSlice.actions;
        // 导出 reducer 给 store 使用
        export default authSlice.reducer;
        ```
      **注意点**：
      1. 声明修改函数时，放在reducers里，默认传入state与action参数（如果你好奇为什么会有action参数 可参考useReducer hook ）但如果不使用某个，请不要声明
      2. 声明异步函数时，注意在slice外部声明，slice内部只需要根据extraReducers里addCase的状态，给出对应操作即可，不要传入该函数，你可以理解为对于异步操作，只监听状态变化
      3. 每个同步操作都要导出才能使用，不能直接使用你声明的名字，这些只是Slice的参数 reducer整体要导出给store使用
   3. 注入Provider 依旧在main.tsx 
      ```tsx
      import { Provider } from "react-redux";
      import { store } from './app/store.ts'
      createRoot(document.getElementById('root')!).render(
        <Provider store={store}>
          <App />
        </Provider>
      )
      ```
   4. 在组件中使用useDispatch 来发送动作
      ```tsx
      import { useDispatch } from "react-redux";
      import { login } from "@/features/auth/authSlice";
      import { AppDispatch } from "@/app/store";
      // 获取 dispatch 函数
      const dispatch = useDispatch<AppDispatch>();
      // 触发 login 这个异步 action
      // unwrap() 是 Redux Toolkit 提供的语法糖，让我们可以直接捕获 Promise 的成功/失败
      await dispatch(
        login({
          username: values.account,
          password: values.password,
        })
      ).unwrap();
      ```
   5. 使用useSelector来获取数据
      ```tsx
      import { useSelector } from "react-redux";
      import { RootState } from "@/app/store";
      // 从 auth 模块中读取 isAuthenticated 状态
      // 当 store 中的这个值发生变化时，组件会自动重新渲染
      const { isAuthenticated, user } = useSelector((state: RootState) => state.auth);
      ```

# Axios 
1. 版本号^1.13.2
2. 作用是处理与后端的 HTTP 通信，支持 Promise API，拦截请求与响应，自动转换 JSON 数据等。相比原生的 `fetch`，它提供了更强大的配置与拦截器功能。
