# React Router
1. 版本号^6.20.1
2. 作用是在单页应用实现页面导航与路由管理 无需每次请求新页面（局部刷新）
3. 具体如何使用可参考对应文档 这里只介绍功能与使用技巧
4. 注意点：
   - `BrowserRouter`作为路由上下文声明与使用在`main.tsx` ，这也是上下文相关声明的地方 后续应当遵守
   - 我们使用组件声明式的路由实现（`<Routes>`） 在最新版有配置对象式的实现`const routes = useRoutes（{}）` 但这种太复杂了 如果你有兴趣后续可改成这种 
   - 注意只有一个`Routes` 其余为`Route` 你可以使用自闭标签（`<Route/>`）与常规标签(`<Route><Route/>`)来声明 但嵌套路由的外层路由只能是常规标签 包裹子路由
   - 不要再使用`<a/>` 后续只使用`<Link/>` 注意这个与`<NavLink/>`区别 后者用于导航 会给当前使用的导航一个`active`的类名，但是`tailwindCss`不是面向类名的 所以我们的组件使用如下
     ```tsx 
     <NavLink to={to} end={end}>
       {({ isActive }) => (
         <div
           className={`
             flex items-center gap-1
             px-4 py-2 rounded-full
             transition-all duration-400 relative
             hover:bg-slate-300
             ${
               isActive
                 ? "bg-accent ring-1 ring-border text-foreground font-medium shadow"
                 : "text-muted-foreground hover:bg-muted/100 hover:text-foreground"
             }
           `}
         >
           <Icon
             strokeWidth={isActive ? 2.5 : 1.5}
             size={36}
             className="h-4 w-4"
           />
           <span
             className={`whitespace-nowrap ${
               isActive ? "font-bold" : "font-normal"
             }`}
           >
             {label}
           </span>
           {isActive && (
             <span
               className="
           absolute left-2 right-2 -bottom-1
           h-0.5
           rounded-full
           bg-primary
         "
             />
           )}
         </div>
       )}
     </NavLink>
     ```
    使用特殊技巧 当函数作为`<NavLink/>`的子元素时 会自动传入参数`isActive`来动态赋予CSS属性 `end`为`true`时 精确匹配才会赋予`active`属性
5. 实际作用 
   - 将特定的URL映射到对应的React组件
   - 管理浏览器历史记录 （nav（-1）是跳转上一个界面）
6. 使用的功能特性 
   - 嵌套路由
     ```tsx 
     <Route element={<AuthLayout />}>
       <Route path="/login" element={<LoginPage />}></Route>
       <Route path="/signup" element={<SignupPage />}></Route>
       <Route path="/forget" element={<ForgetPage />}></Route>
     </Route>
     ```
    当条件匹配时 子路由会渲染到父路由组件`<Outlet/>`对应的位置
    注意 这里外层`Route`没有给出路径 说明这是一个纯粹的父路由不会单独出现，只在url匹配其子路由时渲染
    若给出路径 则可能出现只渲染父路由的情况 此时`<Outlet/>`渲染为空
    `/`开头的`path`表示这是一个绝对路径，若没有则是相对路径（一般用在父路由给出路径的情况下）相对父路由
    - 路径导航
     ```tsx 
     <Route index element={<Navigate to="/home" />}></Route>
     const nav = useNavigate();
     nav(-1)
     ```
    以上分别是声明式导航 使用`<Navigate/>`组件 一旦渲染 立即跳转 与 函数式导航 使用`nav`函数导航
    - 动态路由
     ```tsx 
     <Route path=":id" element={<SubmissionPage/>}/>
     ```
    使用`：id`表示后续参数任意 但全部会作为`id`参数 可在组件内部使用`useParams`读取 用于传递信息
